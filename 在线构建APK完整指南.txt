方案一：GitHub在线构建（推荐 - 完全免费，无需安装软件）
=================================================================

【步骤1：准备GitHub账号】
1. 访问 https://github.com
2. 如果没有账号，点击"Sign up"注册
3. 登录GitHub

【步骤2：创建新仓库】
1. 点击右上角的"+"号
2. 选择"New repository"
3. 填写信息：
   - Repository name: ESP32-Debugger-App
   - Description: ESP32 S3 ZERO调试助手
   - 选择"Public"（公开）
   - 不要勾选"Initialize this repository with a README"
4. 点击"Create repository"

【步骤3：上传项目文件】

方法A：使用GitHub网页上传（简单）
1. 在新创建的仓库页面，点击"uploading an existing file"
2. 将整个项目文件夹拖拽到页面
   或点击"choose your files"选择文件
3. 等待上传完成
4. 在底部填写：
   - Commit message: "Initial commit"
5. 点击"Commit changes"

方法B：使用Git命令行（如果熟悉Git）
打开命令提示符，执行：

cd C:\Users\wd_al\Desktop\ESP32_Debugger_App
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/ESP32-Debugger-App.git
git push -u origin main

【步骤4：触发自动构建】
1. 上传完成后，GitHub会自动开始构建
2. 点击仓库顶部的"Actions"标签
3. 查看构建进度（大约需要5-10分钟）
4. 等待显示绿色✓表示构建成功

【步骤5：下载APK】
1. 在Actions页面，点击最新的构建任务
2. 向下滚动到"Artifacts"部分
3. 点击"ESP32-Debugger-APK"下载
4. 解压下载的zip文件
5. 得到app-debug.apk文件

【步骤6：安装到手机】
1. 将app-debug.apk传输到OnePlus 8T
2. 在手机上找到APK文件
3. 点击安装
4. 如果提示"未知来源"，在设置中允许
5. 安装完成！

方案二：使用Android Studio本地构建
=================================================================

【步骤1：下载Android Studio】
1. 访问 https://developer.android.com/studio
2. 下载Windows版本（约1GB）
3. 运行安装程序
4. 选择"Standard"安装类型
5. 等待下载SDK组件（约3-5GB）

【步骤2：打开项目】
1. 启动Android Studio
2. 选择"Open"
3. 浏览到：C:\Users\wd_al\Desktop\ESP32_Debugger_App
4. 点击"OK"

【步骤3：等待同步】
1. Android Studio会自动同步Gradle
2. 首次同步需要10-20分钟
3. 等待底部状态栏显示"Gradle sync finished"

【步骤4：构建APK】
1. 点击菜单：Build -> Build Bundle(s) / APK(s) -> Build APK(s)
2. 等待构建完成（3-5分钟）
3. 点击弹出通知中的"locate"
4. APK位置：app\build\outputs\apk\debug\app-debug.apk

【步骤5：安装到手机】
方法A：通过USB直接安装
1. 用USB线连接手机和电脑
2. 在手机上启用"开发者选项"和"USB调试"
3. 在Android Studio中点击绿色的"Run"按钮▶
4. 选择你的设备
5. APK自动安装到手机

方法B：手动安装
1. 将APK复制到手机
2. 在手机上点击APK文件安装

常见问题
=================================================================

Q: GitHub Actions构建失败怎么办？
A: 1. 检查Actions页面的错误日志
   2. 确保所有文件都已上传
   3. 检查gradle配置文件是否正确

Q: 下载的APK无法安装？
A: 1. 确保手机允许"未知来源"安装
   2. 检查Android版本（需要8.0+）
   3. 尝试重新下载APK

Q: 构建时间太长？
A: GitHub Actions首次构建需要下载依赖，
   后续构建会使用缓存，速度会快很多

Q: 可以修改代码后重新构建吗？
A: 可以！修改代码后重新上传到GitHub，
   或在Android Studio中修改后重新构建
